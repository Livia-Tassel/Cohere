## 第一步：部署后端到 Render（5 分钟）

### 1. 打开 Render 并登录
- 访问：https://render.com
- 点击右上角 **"Sign Up"** 或 **"Log In"**
- 选择 **"Sign in with GitHub"**
- 授权 Render 访问你的 GitHub 账号

### 2. 创建 Web Service
- 点击 **"New +"** 按钮
- 选择 **"Web Service"**
- 在仓库列表中找到 **"Cohere"**
- 点击 **"Connect"**

### 3. 配置服务

填写以下信息：

```
Name: cohere-api
Region: Singapore (或选择离你最近的区域)
Branch: main
Root Directory: server
Runtime: Node
Build Command: npm install
Start Command: npm start
Instance Type: Free
```

### 4. 添加环境变量

点击 **"Advanced"** 按钮，然后点击 **"Add Environment Variable"**，添加以下变量：

```
PORT
5000

MONGODB_URI
mongodb+srv://livia:20041014@cohere.i32mvls.mongodb.net/?appName=Cohere

JWT_SECRET
34cfbdc028549964a4c94c6a7daf65ad315da32e95d5aa0dc0c04ea7a7a86f70ad18d02ea01bb5039bc7c250139cf94506ba111351350f2c6d1ac5b3c893984a

NODE_ENV
production
```

### 5. 创建服务
- 点击 **"Create Web Service"**
- 等待部署完成（约 3-5 分钟）
- 部署成功后，你会看到一个绿色的 **"Live"** 标志

### 6. 复制 API URL
在页面顶部，你会看到类似这样的 URL：
```
https://cohere-api-xxxx.onrender.com
```

**📝 记下这个 URL，下一步需要用到！**

## 第二步：部署前端到 Vercel（5 分钟）

### 1. 打开 Vercel 并登录
- 访问：https://vercel.com
- 点击 **"Sign Up"** 或 **"Log In"**
- 选择 **"Continue with GitHub"**
- 授权 Vercel 访问你的 GitHub 账号

### 2. 导入项目
- 点击 **"Add New..."** → **"Project"**
- 在仓库列表中找到 **"Cohere"**
- 点击 **"Import"**

### 3. 配置项目

填写以下信息：

```
Framework Preset: Vite
Root Directory: client
Build Command: npm run build
Output Directory: dist
Install Command: npm install
```

### 4. 添加环境变量

点击 **"Environment Variables"**，添加：

```
Name: VITE_API_URL
Value: https://cohere-api-xxxx.onrender.com/api
```

⚠️ **重要**：
- 使用你在第一步获得的 Render URL
- 记得在 URL 后面加上 `/api`
- 例如：`https://cohere-api-xxxx.onrender.com/api`

### 5. 部署
- 点击 **"Deploy"**
- 等待部署完成（约 2-3 分钟）
- 部署成功后，你会看到庆祝动画 🎉

### 6. 获取网站 URL
部署完成后，你会看到类似这样的 URL：
```
https://cohere-xxxx.vercel.app
```

**📝 记下这个 URL！**

---

## 第三步：更新 CORS 配置（2 分钟）

为了让前端能够正常访问后端 API，需要更新 CORS 配置。

### 1. 回到 Render Dashboard
- 访问：https://dashboard.render.com
- 点击你的 **"cohere-api"** 服务

### 2. 添加 CORS 环境变量
- 点击左侧的 **"Environment"** 标签
- 点击 **"Add Environment Variable"**
- 添加：
  ```
  Name: CORS_ORIGIN
  Value: https://cohere-xxxx.vercel.app
  ```
  （使用你的 Vercel URL，不要加 /api）

### 3. 保存并重新部署
- 点击 **"Save Changes"**
- Render 会自动重新部署（约 2-3 分钟）
- 等待状态变为 **"Live"**

## 🐛 常见问题解决

### 问题 1：网站打开但显示空白

**原因**：前端无法连接到后端 API

**解决方案**：
1. 检查 Vercel 的 `VITE_API_URL` 是否正确
2. 确保 URL 以 `/api` 结尾
3. 检查 Render 服务是否在运行（Live 状态）
4. 打开浏览器控制台（F12）查看错误信息

### 问题 2：CORS 错误

**错误信息**：
```
Access to XMLHttpRequest has been blocked by CORS policy
```

**解决方案**：
1. 确认 Render 的 `CORS_ORIGIN` 环境变量已添加
2. 确认值是你的 Vercel URL（不带 /api）
3. 等待 Render 重新部署完成（2-3 分钟）
4. 清除浏览器缓存并刷新

### 问题 3：502 Bad Gateway

**原因**：后端服务未启动或正在部署

**解决方案**：
1. 等待 3-5 分钟（Render 可能正在部署）
2. 检查 Render 日志是否有错误
3. 确认环境变量都已正确添加
4. 如果是免费版，可能需要等待服务唤醒（30 秒）

### 问题 4：无法注册或登录

**原因**：数据库连接问题

**解决方案**：
1. 检查 `MONGODB_URI` 是否正确
2. 登录 MongoDB Atlas 确认集群在运行
3. 检查 Network Access 是否允许所有 IP（0.0.0.0/0）
4. 查看 Render 日志中的数据库连接错误

### 问题 5：Render 服务一直显示 "Deploying"

**解决方案**：
1. 等待 5-10 分钟（首次部署较慢）
2. 检查 Logs 标签是否有错误
3. 确认 Root Directory 设置为 `server`
4. 确认 Build Command 是 `npm install`
5. 确认 Start Command 是 `npm start`
